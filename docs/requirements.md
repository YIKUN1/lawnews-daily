# 热门法律事件收集系统 - 需求文档

## 1. 项目概述

### 1.1 项目名称
LawNews Daily（法律日报收集系统）

### 1.2 项目目标
自动收集每日热门法律事件，通过AI归纳整理后，推送到微信群，帮助用户高效获取法律资讯。

### 1.3 目标用户
- 法律从业者
- 法律爱好者
- 需要关注法律动态的普通用户

---

## 2. 功能需求

### 2.1 数据采集模块

#### 2.1.1 数据来源
| 来源类型 | 具体网站 | 采集方式 | 优先级 |
|---------|---------|---------|--------|
| 官方法律网站 | 中国法院网、最高法官网、中国裁判文书网 | 爬虫/RSS | 高 |
| 社交媒体热搜 | 微博热搜、知乎热榜 | API/爬虫 | 高 |
| 新闻门户 | 新浪法律、网易法治、腾讯新闻法律频道 | RSS/爬虫 | 中 |
| 微信公众号 | 法律类公众号（通过RSSHub） | RSS | 中 |

#### 2.1.2 采集内容
- 标题
- 摘要/正文片段
- 来源链接
- 发布时间
- 热度指标（如有）

#### 2.1.3 采集频率
- 每天采集2次（早报、晚报）
- 支持手动触发采集

### 2.2 内容处理模块

#### 2.2.1 关键词过滤
法律相关关键词库：
- 案件类型：民事、刑事、行政、知识产权、劳动争议等
- 法律术语：判决、诉讼、法规、条文、裁定、起诉等
- 主体：法院、检察院、律师、当事人、被告、原告等

#### 2.2.2 去重处理
- URL去重：相同链接不重复采集
- 内容去重：相似度超过80%视为重复
- 时间窗口：7天内已推送内容不再推送

#### 2.2.3 热度排序
- 综合社交平台热度
- 来源权重（官网 > 社交媒体 > 新闻门户）
- 时效性权重

### 2.3 AI摘要模块

#### 2.3.1 支持的AI服务
- 通义千问（推荐，国内稳定）
- OpenAI GPT
- 文心一言
- DeepSeek

#### 2.3.2 摘要要求
- 每条事件生成50-100字摘要
- 提取关键法律要点
- 保持客观中立
- 格式统一

#### 2.3.3 输出格式
```
【标题】xxx
【要点】xxx
【来源】xxx
【链接】xxx
```

### 2.4 推送模块

#### 2.4.1 支持的推送方式
| 方式 | 适用场景 | 配置要求 |
|-----|---------|---------|
| 企业微信机器人 | 企业微信群 | Webhook URL |
| PushPlus | 个人微信/群 | Token |
| Wechaty | 任意微信群 | 闲置微信号 |

#### 2.4.2 推送时间
- 早报：08:00
- 晚报：18:00
- 支持自定义时间

#### 2.4.3 推送内容格式
```
⚖️ 法律日报 - 2024年3月1日 早报

━━━━━━━━━━━━━━━

📌 热门事件

1️⃣ 【标题】xxx
   【要点】xxx
   【来源】xxx

2️⃣ ...

━━━━━━━━━━━━━━━
共收录 N 条法律资讯
```

### 2.5 定时任务模块

#### 2.5.1 调度方式
- 本地：APScheduler
- 服务器：Cron / Systemd Timer

#### 2.5.2 任务管理
- 支持启动/停止/重启
- 任务执行日志
- 失败重试机制（最多3次）

---

## 3. 非功能需求

### 3.1 性能要求
- 单次采集时间 < 5分钟
- 单次推送内容条数：5-10条
- 系统资源占用 < 500MB内存

### 3.2 可靠性要求
- 采集失败自动跳过，不影响其他来源
- 推送失败自动重试（3次）
- 数据本地缓存，防止丢失

### 3.3 安全性要求
- API密钥加密存储
- 不存储敏感用户信息
- 遵守各网站robots.txt

### 3.4 可维护性要求
- 模块化设计，便于扩展新来源
- 配置文件集中管理
- 完善的日志系统

---

## 4. 系统约束

### 4.1 技术约束
- 开发语言：Python 3.8+
- 运行环境：Windows / Linux
- 依赖最小化原则

### 4.2 法律约束
- 遵守著作权法，仅采集公开信息
- 尊重网站服务条款
- 不进行商业分发
